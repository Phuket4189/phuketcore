# PhuketCore 网站设计方案

## 一、项目概述

**项目名称**: PhuketCore 个人博客网站  
**项目类型**: 个人博客 + 项目展示  
**开发模式**: 前后端分离  
**部署方式**: Node.js 服务器 + Vue SPA

---

## 二、设计思路

### 2.1 整体架构设计

采用 **前后端分离** 的架构模式，前端使用 Vue 3 构建单页应用（SPA），后端使用 Express 提供 API 服务和静态文件托管。

```
┌─────────────────────────────────────────┐
│         前端 (Vue 3 + Vite)         │
│  - 首页展示                          │
│  - 开发日志                          │
│  - 项目详情页                        │
│  - 后台管理                          │
└─────────────────────────────────────────┘
              ↓ HTTP/API
┌─────────────────────────────────────────┐
│       后端 (Node.js + Express)         │
│  - 静态文件服务                      │
│  - API 接口                         │
│  - 文档读写                          │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│         数据存储 (文件系统)             │
│  - public/docs/                      │
│    - devlog.md                       │
│    - BSL-README.md                   │
│    - 其他文档文件                     │
└─────────────────────────────────────────┘
```

### 2.2 页面布局设计

#### 2.2.1 整体布局
- **左侧侧边栏**: 固定宽度 220px，包含导航菜单
- **右侧内容区**: 自适应宽度，使用 `flex: 1 1 auto` 填充剩余空间
- **响应式设计**: 移动端适配（待完善）

#### 2.2.2 首页设计
- **Hero 区域**: 全屏高度（100vh），蓝白瀑布渐变背景
  - 标题: PhuketCore，大字号（3.2rem），字重 800
  - 副标题: 编程玩家个人博客
  - 滚动按钮: 圆形按钮，引导用户向下滚动
- **项目展示区**: 
  - 项目卡片: 渐变背景，悬停效果
  - 功能列表: 列表形式展示
  - 技术栈标签: 底部展示
- **动画效果**: 
  - Hero 内容聚合渐显动画
  - 项目卡片滚动进入动画（IntersectionObserver）

#### 2.2.3 项目详情页（BSL）
- **Hero 区域**: 与首页一致的蓝白瀑布渐变背景
  - 项目标题: BlockServerLauncher
  - 副标题: 一体化 Minecraft 服务器管理启动器
  - 技术标签: Python、PySide6、跨平台、开源
- **内容区域**:
  - 项目概述 + 数据统计（10+ 功能模块、72 细分功能、3 支持平台）
  - 核心功能: 6 个功能卡片，网格布局
  - 目标用户: 4 个用户类型卡片
  - 技术栈: 4 项技术信息
  - 相关链接: 按钮形式

#### 2.2.4 开发日志页
- **Markdown 渲染**: 使用 marked 库将 Markdown 转换为 HTML
- **样式优化**: 自定义 Markdown 样式，与网站整体风格统一
- **动画效果**: 页面进入动画

#### 2.2.5 后台管理页
- **文档列表**: 展示 public/docs 目录下的所有文件
- **Markdown 编辑器**: 实时预览的编辑器
- **文件操作**: 读取、编辑、保存文档

### 2.3 视觉设计原则

1. **现代化设计**: 
   - 卡片式布局
   - 圆角设计（8-16px）
   - 柔和阴影（box-shadow）
   - 渐变背景

2. **色彩方案**:
   - 主色调: 蓝色系（#3b82f6、#2563eb）
   - 背景: 蓝白瀑布渐变（#e3eafe → #f5f5fa）
   - 文字: 深灰色系（#2d3748、#4a5568、#64748b）
   - 强调色: 橙色（#f59e0b）

3. **字体排版**:
   - 标题: 字重 700-800，字间距 1-2px
   - 正文: 行高 1.6-1.8，颜色 #64748b
   - 统一字体: system-ui

4. **交互体验**:
   - 悬停效果: transform、box-shadow 变化
   - 过渡动画: 0.2-0.3s ease
   - 滚动动画: IntersectionObserver 实现元素进入动画

## 四、开发难点与解决方案

### 4.1 开发环境 CORS 问题

**问题描述**:  
在开发模式下，前端运行在 Vite 开发服务器（通常端口 5173），后端 API 运行在 Express 服务器（端口 3000），直接请求会触发 CORS 跨域错误。

**解决方案**:  
在 `vite.config.js` 中配置代理，将 `/api` 请求代理到后端服务器：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:3000',
      changeOrigin: true,
      rewrite: path => path.replace(/^\/api/, '/api')
    }
  }
}
```

### 4.2 SPA 路由刷新 404 问题

**问题描述**:  
部署后，直接访问 `/dlog` 或刷新页面会返回 404，因为服务器找不到对应的文件。

**解决方案**:  
在 Express 中配置 SPA 回退路由，将未匹配的 GET 请求返回 `index.html`：

```javascript
app.use((req, res, next) => {
  if (req.method === 'GET' && !req.url.startsWith('/api/')) {
    const index = path.join(__dirname, 'dist', 'index.html')
    if (fs.existsSync(index)) {
      return res.sendFile(index)
    }
  }
  next()
})
```

### 4.3 Markdown 样式统一

**问题描述**:  
使用 `marked` 渲染的 Markdown HTML 没有样式，与网站整体风格不匹配。

**解决方案**:  
在组件中自定义 Markdown 样式，使用 `:deep()` 选择器覆盖默认样式：

```css
.doc-content :deep(h1) {
  font-size: 2rem;
  color: #1e293b;
}
.doc-content :deep(p) {
  line-height: 1.8;
  color: #64748b;
}
```

### 4.4 文件路径安全

**问题描述**:  
文档 API 需要读写文件系统，存在路径遍历安全风险。

**解决方案**:  
对文件名进行正则验证，并检查路径是否在允许的目录内：

```javascript
if (!/^[\w.\-]+$/.test(name)) return res.status(400).json({ error: 'invalid_name' })
const p = path.join(DOC_DIR, name)
if (!p.startsWith(DOC_DIR)) return res.status(400).json({ error: 'invalid_path' })
```

### 4.5 动画性能优化

**问题描述**:  
多个元素同时执行动画可能导致页面卡顿。

**解决方案**:  
使用 CSS `transform` 和 `opacity` 属性实现动画，避免触发重排：

```css
.fadeup-enter-from {
  opacity: 0;
  transform: translateY(40px) scale(0.96);
}
```

使用 `IntersectionObserver` 实现懒加载动画，只对进入视口的元素执行动画。

### 4.6 响应式布局适配

**问题描述**:  
固定宽度的侧边栏在移动端会挤压内容区域。

**解决方案**:  
使用媒体查询，在小屏幕上调整布局：

```css
@media (max-width: 768px) {
  .content {
    margin-left: 0;
  }
  .sidebar {
    display: none;
  }
}
```

### 4.7 开发与生产环境切换

**问题描述**:  
开发环境和生产环境需要不同的配置（API 地址、静态文件路径等）。

**解决方案**:  
- 开发环境: 使用 `npm run dev`，Vite 代理 API
- 生产环境: 使用 `node server.js`，Express 同时提供 API 和静态文件

## 五、部署方案

### 5.1 开发环境

```bash
npm install
npm run dev
```

访问 `http://localhost:5173`，Vite 自动代理 `/api` 请求到 Express 服务器。

### 5.2 生产环境

```bash
npm run build
npm start
```

1. `npm run build` 将前端构建到 `dist/` 目录
2. `npm start` 启动 Express 服务器，提供 API 和静态文件服务
3. 访问 `http://localhost:3000` 或部署到服务器

### 5.3 服务器部署建议

1. **使用 PM2 管理进程**:
```bash
npm install -g pm2
pm2 start server.js --name phuketcore
pm2 startup
pm2 save
```

2. **使用 Nginx 反向代理**:
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 七、总结

PhuketCore 网站采用现代化的前后端分离架构，使用 Vue 3 + Express 技术栈，实现了个人博客、项目展示、开发日志和后台管理等功能。设计上注重用户体验，采用卡片式布局、渐变背景、流畅动画等现代设计元素。

开发过程中解决了 CORS 跨域、SPA 路由、文件安全、动画性能等技术难点，为后续功能扩展奠定了良好基础。
